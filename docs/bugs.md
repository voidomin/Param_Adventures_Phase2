# Bug Tracker - Param Adventure Phase 2

Track all bugs, issues, and UI defects found during development.

## How to Use

- Add bugs with the format: `BUG-XXX`
- Set **Priority**: 🔴 Critical | 🟡 Medium | 🟢 Low
- Set **Status**: 🆕 Open | 🔧 In Progress | ✅ Fixed | ❌ Won't Fix
- Always note the **date found** and **the commit that fixed it**

## Reporting a New Bug

When you find a bug, add it to the **Active Bugs** table above with:

1. **ID**: Next sequential `BUG-XXX` number
2. **Priority**: How badly it affects the user experience
3. **Description**: What happens vs what should happen
4. **Steps to reproduce** (add below the table if complex):

```
### BUG-XXX: [Title]
**Steps to reproduce:**
1. Go to [page]
2. Click [button]
3. Observe [broken behavior]

**Expected:** [what should happen]
**Actual:** [what happens instead]
**Screenshot:** [if applicable]
```
